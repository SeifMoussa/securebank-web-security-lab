# Contributing

This repository is a fictional lab project for defensive secure software engineering practice.

## Ground Rules

- Do not add real customer data.
- Do not add real banking identifiers.
- Do not add exploit tooling.
- Keep adversarial inputs harmless and scoped to local defensive tests.
- Keep the application server-rendered.

## Local Checks

Before proposing changes, run:

```bash
make lint
make test
```
